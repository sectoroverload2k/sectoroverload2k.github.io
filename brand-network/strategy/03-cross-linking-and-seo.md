# Cross-Linking & SEO

## Principle

anthonyl.com **sends leads down** to the commercial sites; the commercial sites **send
authority/trust back up** to Anthony. Do it with **contextual, role-appropriate links** — not
identical footer link-blocks repeated across every domain, which can read to Google as a link
scheme and look spammy to humans. A small shared "Triton network" bar is fine; the high-value
links are the in-content contextual ones.

## The cross-link map

| From → To | How it appears | Purpose |
|---|---|---|
| anthonyl.com → choosetriton | On each tutorial: *"Need this done for your team? I do this through **Triton** →"* + an "Advisory" nav item | Convert readers into consulting leads (the money path) |
| anthonyl.com → agency / processing | Project pages deep-link the relevant spoke (*"the AI work powering this → Triton Agency"*) | Route by intent |
| choosetriton → anthonyl.com | *"Founded by **Anthony Linsday**, AWS Solutions Architect, 20+ yrs"* → About | Borrow authority + a human face = trust |
| choosetriton ↔ processing / agency | Hub service cards deep-link each spoke; spokes show *"Part of the Triton network"* back to hub | Hub-and-spoke routing + land-and-expand |
| choosetriton → PazNetworks | "Partners" section: network/phone/hosting by PazNetworks (reciprocal link back) | Sell Paul's line as part of the offer |
| all spokes → choosetriton | Light shared "Triton network" brand bar in header | Tie the family together |

## Founder-led SEO flywheel (the channel Paul can't match)

1. Anthony writes deep technical tutorials (Hairpin NAT, Terraform, AWS, email deliverability,
   multi-site cloud) targeting **high-intent keywords** the buyers actually search.
2. Organize them into **topic clusters** (AWS / Networking / Automation / Payments-adjacent),
   interlinked, to build domain authority — this is what outranks single-page competitors.
3. Each article ends with a **contextual CTA into Triton** (free audit / advisory).
4. Founder-written content earns more backlinks and trust than corporate copy — Anthony's name
   and face are the asset.

## Technical SEO hygiene (apply to all sites)
- Unique, descriptive `<title>` and meta description per page (no keyword stuffing — the legacy
  anthonyl.com keyword-stuffed meta should be removed).
- One `<h1>` per page; semantic headings.
- Fast, mobile-first, accessible (alt text, contrast).
- `schema.org` markup: `Person` (Anthony), `Organization` (each Triton entity),
  `Article`/`HowTo` (tutorials), `Service` (offerings), `BreadcrumbList`.
- XML sitemaps + clean internal linking within each domain.
- Consistent NAP (name/address/phone) for local/again-trust signals.
